# 機能定義書：画像投稿機能

## 1. 業務フロー

### 1-1. 画像添付フロー（投稿作成時）
```
投稿作成モーダルで画像アイコン（カメラ）をクリック
    │
OS のファイル選択ダイアログを表示
    │
画像ファイルを選択（複数選択可、最大 4 枚）
    │
    ├── 5 枚以上選択 → 「画像は最大 4 枚まで添付できます」を表示
    ├── 非対応フォーマット → 「JPEG・PNG のみ対応しています」を表示
    ├── ファイルサイズ超過 → 「1 枚あたり最大 10MB です」を表示
    │
    └── バリデーション成功
            │
        モーダル内にプレビューを表示（4 枚グリッド）
            │
        「×」ボタンで個別に削除可能
            │
        「投稿」ボタンで投稿と同時に S3 にアップロード
```

### 1-2. 画像アップロードフロー（バックエンド）
```
フロントエンドから画像ファイルを multipart/form-data で受信
    │
バックエンドが S3 にアップロード
    │
S3 の URL を取得
    │
post_images テーブルに URL と表示順序を保存
    │
投稿レスポンスに画像 URL リストを含めて返す
```

### 1-3. 画像削除フロー（投稿削除時）
```
投稿削除リクエストを受信
    │
post_images テーブルから画像 URL を取得
    │
S3 から画像ファイルを削除
    │
post_images レコードを削除
    │
posts レコードを削除
```

## 2. ユースケース

### UC-14: 画像添付投稿

| 項目 | 内容 |
|---|---|
| UC-ID | UC-14 |
| ユースケース名 | 画像添付投稿 |
| アクター | 認証済みユーザー |
| 事前条件 | ログイン済み、投稿作成モーダルを表示中 |
| 事後条件 | 投稿と画像が DB および S3 に保存され、タイムラインに表示される |

**基本フロー:**
1. 投稿作成モーダルのカメラアイコンをクリックする
2. ファイル選択ダイアログで画像を選択する（1〜4 枚）
3. モーダル内にプレビューが表示される
4. テキストを入力する（任意）
5. 「投稿」ボタンをクリックする
6. バックエンドが S3 にアップロードし、URL を DB に保存する
7. タイムラインに画像付き投稿が表示される

**代替フロー（バリデーションエラー）:**
- 5 枚以上選択時：「画像は最大 4 枚まで添付できます」を表示する
- 非対応フォーマット時：「JPEG・PNG のみ対応しています」を表示する
- サイズ超過時：「1 枚あたり最大 10MB です」を表示する

---

### UC-15: タイムラインでの画像表示

| 項目 | 内容 |
|---|---|
| UC-ID | UC-15 |
| ユースケース名 | タイムラインでの画像表示 |
| アクター | 認証済みユーザー |
| 事前条件 | ログイン済み、画像付き投稿が存在する |
| 事後条件 | タイムラインおよび投稿詳細画面に画像が表示される |

**基本フロー:**
1. タイムライン画面を表示する
2. 画像付き投稿カードに画像グリッドが表示される
   - 1 枚：全幅表示
   - 2 枚：横 2 分割
   - 3 枚：左 1 枚 + 右 2 段
   - 4 枚：2×2 グリッド
3. 投稿詳細画面でも同様のレイアウトで表示される

## 3. 機能要件

### 画像アップロード

- [ ] 対応フォーマット：JPEG・PNG
- [ ] 1 投稿あたり最大 4 枚まで添付可能
- [ ] 1 枚あたりの最大ファイルサイズは 10MB
- [ ] 画像は AWS S3 バケットに保存し、URL を post_images テーブルで管理する
- [ ] S3 への直接アクセスは禁止し、IAM ロール経由でのみバックエンドからアクセスする
- [ ] ファイル名は衝突を防ぐため UUID でリネームして保存する（例：`posts/{postId}/{uuid}.jpg`）

### 画像表示

- [ ] タイムライン・投稿詳細画面の両方で画像を表示する
- [ ] 画像枚数に応じてグリッドレイアウトを変える（1/2/3/4 枚パターン）
- [ ] 画像クリックで拡大表示（モーダル）できる

### 画像削除

- [ ] 投稿削除時に S3 の画像ファイルも連動削除する
- [ ] 投稿作成モーダル内でのプレビュー中に「×」ボタンで個別削除できる

## 4. API 設計（概要）

| メソッド | エンドポイント | 説明 | 認証 |
|---|---|---|---|
| POST | `/api/posts` | 投稿作成（画像は multipart/form-data で同送） | 必要 |

※ 画像は投稿作成 API と一体で送信する。独立した画像アップロードエンドポイントは設けない。

### S3 ファイルパス規則

```
posts/{postId}/{uuid}.{ext}
avatars/{userId}/{uuid}.{ext}
```
