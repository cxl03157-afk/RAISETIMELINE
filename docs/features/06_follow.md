# 機能定義書：フォロー・フォロワー・ユーザー検索機能

## 1. 業務フロー

### 1-1. ユーザー検索フロー
```
ナビゲーションバーの検索アイコンをクリック
    │
ユーザー検索画面へ遷移
    │
検索バーにユーザー名または表示名を入力
    │
300ms デバウンス後にバックエンドへ検索リクエスト
    │
    ├── 検索結果あり → ユーザーカード一覧を表示
    │       各カード：アバター・表示名・@username・フォロー状態・フォローボタン
    │
    └── 検索結果なし → 「ユーザーが見つかりませんでした」を表示
```

### 1-2. フォローフロー
```
ユーザー検索画面またはプロフィール画面で「フォローする」ボタンをクリック
    │
    ├── 未ログイン → ログイン画面へリダイレクト（※通常はログイン済みのみ操作可）
    │
    └── ログイン済み
            │
        バックエンドに follows レコードを追加
            │
        ボタンが「フォロー中」に変わる
            │
        対象ユーザーのフォロワー数を +1 して表示を更新
```

### 1-3. フォロー解除フロー
```
「フォロー中」ボタンをクリック（またはホバー時に「フォロー解除」に変化）
    │
バックエンドから follows レコードを削除
    │
ボタンが「フォローする」に戻る
    │
対象ユーザーのフォロワー数を -1 して表示を更新
```

### 1-4. フォロー中一覧表示フロー
```
プロフィール画面の「フォロー中 N」をクリック
    │
フォロー中一覧画面へ遷移
    │
フォローしているユーザーのカード一覧を表示
    │
各カードの「フォロー中」ボタンでフォロー解除可能
```

### 1-5. フォロワー一覧表示フロー
```
プロフィール画面の「フォロワー N」をクリック
    │
フォロワー一覧画面へ遷移
    │
フォロワーユーザーのカード一覧を表示
    │
各カードの「フォローする」/「フォロー中」ボタンで相互フォロー操作可能
```

## 2. ユースケース

### UC-16: ユーザー検索

| 項目 | 内容 |
|---|---|
| UC-ID | UC-16 |
| ユースケース名 | ユーザー検索 |
| アクター | 認証済みユーザー |
| 事前条件 | ログイン済み |
| 事後条件 | 検索キーワードに一致するユーザーが一覧表示される |

**基本フロー:**
1. ユーザー検索画面の検索バーにキーワードを入力する
2. 300ms 後にバックエンドへリクエストを送信する
3. ユーザー名（`username`）または表示名（`display_name`）で部分一致検索を行う
4. 検索結果としてユーザーカード一覧が表示される（自分自身は除外する）
5. 各カードに現在のフォロー状態（フォローする / フォロー中）が表示される

**代替フロー（結果なし）:**
- 一致するユーザーがいない場合、「ユーザーが見つかりませんでした」を表示する

**検索仕様:**
- 検索対象：`username`（ユーザー名）と `display_name`（表示名）の OR 検索
- マッチング：部分一致（ILIKE `%keyword%`）
- 除外条件：自分自身は検索結果に表示しない
- 件数：最大 20 件表示（ページネーションなし）

### UC-17: フォロー

| 項目 | 内容 |
|---|---|
| UC-ID | UC-17 |
| ユースケース名 | フォロー |
| アクター | 認証済みユーザー |
| 事前条件 | ログイン済み、対象ユーザーをフォローしていない |
| 事後条件 | follows レコードが追加され、フォロー数・フォロワー数が更新される |

**基本フロー:**
1. ユーザー検索画面またはプロフィール画面で「フォローする」ボタンをクリックする
2. バックエンドが follows テーブルにレコードを追加する
3. ボタンが「フォロー中」に変わる
4. フォロワー数が +1 されて表示が更新される

**代替フロー（自己フォロー防止）:**
- 自分自身をフォローしようとした場合、バックエンドは 400 Bad Request を返す
- フロントエンドは自分のプロフィール画面にはフォローボタンを表示しない

## 3. 機能要件

### ユーザー検索

- [ ] ユーザー名（`username`）と表示名（`display_name`）で部分一致検索する
- [ ] 大文字・小文字を区別しない検索（ILIKE）を行う
- [ ] 自分自身は検索結果に表示しない
- [ ] 検索バー入力後 300ms のデバウンスを設ける
- [ ] 最大 20 件を表示する

### フォロー

- [ ] 同一ユーザーを複数回フォローできない（DB UNIQUE 制約で保証）
- [ ] 自分自身をフォローできない（バックエンドで 400 を返す）
- [ ] フォロー / フォロー解除はユーザー検索画面・プロフィール画面・フォロー/フォロワー一覧画面から操作できる

### フォロー / フォロワー一覧

- [ ] プロフィール画面にフォロー中数・フォロワー数を表示する
- [ ] フォロー中一覧・フォロワー一覧はプロフィール画面からアクセスする
- [ ] 他ユーザーのプロフィール画面でもフォロー中数・フォロワー数を閲覧できる

## 4. API 設計（概要）

| メソッド | エンドポイント | 説明 | 認証 |
|---|---|---|---|
| GET | `/api/users/search?q={keyword}` | ユーザー検索 | 必要 |
| POST | `/api/users/{username}/follow` | フォロー | 必要 |
| DELETE | `/api/users/{username}/follow` | フォロー解除 | 必要 |
| GET | `/api/users/{username}/following` | フォロー中一覧取得 | 必要 |
| GET | `/api/users/{username}/followers` | フォロワー一覧取得 | 必要 |
