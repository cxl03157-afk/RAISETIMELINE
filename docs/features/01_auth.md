# 機能定義書：ログイン・認証機能

## 1. 業務フロー

### 1-1. 新規登録フロー
```
ユーザーがアプリにアクセス
    │
新規登録画面を表示
    │
ユーザー情報を入力（ユーザー名・表示名・メールアドレス・パスワード）
    │
「登録する」ボタンをクリック
    │
    ├── バリデーションエラー → エラーメッセージを表示（入力画面に留まる）
    │
    └── バリデーション成功
            │
        DB にユーザーを保存（パスワードは BCrypt でハッシュ化）
            │
        JWT トークンを発行
            │
        タイムライン画面へ遷移
```

### 1-2. ログインフロー
```
ユーザーがログイン画面を表示
    │
メールアドレス・パスワードを入力
    │
「ログイン」ボタンをクリック
    │
    ├── 認証失敗 → 「メールアドレスまたはパスワードが正しくありません」を表示
    │
    └── 認証成功
            │
        JWT トークンを発行
            │
        タイムライン画面へ遷移
```

### 1-3. ログアウトフロー
```
ナビゲーションバーの「ログアウト」をクリック
    │
フロントエンドの JWT トークンを破棄（localStorage から削除）
    │
ログイン画面へ遷移
```

### 1-4. プロフィール編集フロー
```
プロフィール画面で「プロフィールを編集」をクリック
    │
編集モーダルを表示（表示名・自己紹介・アバター画像）
    │
情報を変更して「保存」をクリック
    │
    ├── バリデーションエラー → エラーメッセージを表示
    │
    └── 保存成功 → プロフィール画面を更新して表示
```

## 2. ユースケース

### UC-01: 新規ユーザー登録

| 項目 | 内容 |
|---|---|
| UC-ID | UC-01 |
| ユースケース名 | 新規ユーザー登録 |
| アクター | 未認証ユーザー |
| 事前条件 | アプリにアクセス済み、アカウントを持っていない |
| 事後条件 | アカウントが作成され、JWT トークンが発行される |

**基本フロー:**
1. ユーザーが新規登録画面に遷移する
2. ユーザー名・表示名・メールアドレス・パスワードを入力する
3. 「登録する」ボタンをクリックする
4. バックエンドがバリデーションを行い、DB にユーザーを保存する
5. JWT トークンをレスポンスで返す
6. フロントエンドがトークンを localStorage に保存する
7. タイムライン画面へ遷移する

**代替フロー（バリデーションエラー）:**
- ユーザー名が既に使用済みの場合、「このユーザー名はすでに使用されています」を表示する
- メールアドレスが既に登録済みの場合、「このメールアドレスはすでに登録されています」を表示する
- パスワードが 8 文字未満の場合、「パスワードは 8 文字以上で入力してください」を表示する

---

### UC-02: ログイン

| 項目 | 内容 |
|---|---|
| UC-ID | UC-02 |
| ユースケース名 | ログイン |
| アクター | 未認証ユーザー |
| 事前条件 | アカウントを持っている |
| 事後条件 | JWT トークンが発行され、認証済み状態になる |

**基本フロー:**
1. ユーザーがログイン画面にメールアドレスとパスワードを入力する
2. 「ログイン」ボタンをクリックする
3. バックエンドが認証情報を検証する
4. JWT トークンをレスポンスで返す
5. フロントエンドがトークンを localStorage に保存する
6. タイムライン画面へ遷移する

**代替フロー（認証失敗）:**
- メールアドレスまたはパスワードが誤りの場合、「メールアドレスまたはパスワードが正しくありません」を表示する

---

### UC-03: ログアウト

| 項目 | 内容 |
|---|---|
| UC-ID | UC-03 |
| ユースケース名 | ログアウト |
| アクター | 認証済みユーザー |
| 事前条件 | ログイン済み |
| 事後条件 | JWT トークンが破棄され、未認証状態になる |

**基本フロー:**
1. ナビゲーションバーの「ログアウト」をクリックする
2. フロントエンドが localStorage のトークンを削除する
3. ログイン画面へ遷移する

---

### UC-04: プロフィール編集

| 項目 | 内容 |
|---|---|
| UC-ID | UC-04 |
| ユースケース名 | プロフィール編集 |
| アクター | 認証済みユーザー（自分のプロフィールのみ） |
| 事前条件 | ログイン済み、自分のプロフィール画面を表示中 |
| 事後条件 | 変更内容が DB に保存され、プロフィール画面が更新される |

**基本フロー:**
1. 「プロフィールを編集」ボタンをクリックする
2. 編集モーダルが開く（現在の値がプリセット）
3. 表示名・自己紹介・アバター画像を変更する
4. 「保存」をクリックする
5. バックエンドが DB を更新する
6. モーダルが閉じ、プロフィール画面が更新される

## 3. 機能要件

### 入力バリデーション

| フィールド | ルール |
|---|---|
| ユーザー名 | 必須、3〜20 文字、英数字とアンダースコアのみ、ユニーク |
| 表示名 | 必須、1〜50 文字 |
| メールアドレス | 必須、メール形式、ユニーク |
| パスワード | 必須、8 文字以上 |
| 自己紹介 | 任意、160 文字以内 |

### 認証仕様

- [ ] JWT トークンの有効期限は 24 時間とする
- [ ] パスワードは BCrypt（strength=12）でハッシュ化して保存する
- [ ] ログイン必要な API は Authorization ヘッダー（Bearer トークン）で認証する
- [ ] トークン期限切れ時は 401 を返し、フロントエンドはログイン画面へリダイレクトする

### プロフィール

- [ ] アバター画像は AWS S3 に保存し、URL を DB 管理する
- [ ] アバター画像は JPEG / PNG を許可し、最大ファイルサイズは 5MB とする
- [ ] アバター未設定時はデフォルト画像を表示する

## 4. API 設計（概要）

| メソッド | エンドポイント | 説明 | 認証 |
|---|---|---|---|
| POST | `/api/auth/register` | 新規ユーザー登録 | 不要 |
| POST | `/api/auth/login` | ログイン・JWT 発行 | 不要 |
| GET | `/api/users/{username}` | プロフィール取得 | 必要 |
| PUT | `/api/users/me` | プロフィール更新 | 必要 |
| POST | `/api/users/me/avatar` | アバター画像アップロード | 必要 |
