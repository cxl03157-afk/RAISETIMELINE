# 機能定義書：コメント機能

## 1. 業務フロー

### 1-1. コメント投稿フロー
```
タイムラインまたは投稿詳細画面でコメントアイコンをクリック
    │
投稿詳細画面へ遷移（未遷移の場合）
    │
コメント入力フォームにテキストを入力
    │
「コメントする」ボタンをクリック
    │
    ├── テキストが空 → 「コメントを入力してください」を表示
    │
    └── バリデーション成功
            │
        バックエンドにコメントを送信・保存
            │
        コメント一覧の末尾に新規コメントを追加表示
            │
        入力フォームをリセット
```

### 1-2. コメント削除フロー
```
投稿詳細画面で自分のコメントの「削除」ボタンをクリック
    │
確認ダイアログ「このコメントを削除しますか？」を表示
    │
    ├── 「キャンセル」→ ダイアログを閉じる
    │
    └── 「削除する」→ バックエンドに削除リクエストを送信
                        │
                    コメント一覧から削除されたコメントを非表示にする
                        │
                    投稿のコメント数を更新する
```

## 2. ユースケース

### UC-08: コメント投稿

| 項目 | 内容 |
|---|---|
| UC-ID | UC-08 |
| ユースケース名 | コメント投稿 |
| アクター | 認証済みユーザー（投稿者本人・第三者どちらも可） |
| 事前条件 | ログイン済み、投稿詳細画面を表示中 |
| 事後条件 | コメントが DB に保存され、コメント一覧に表示される |

**基本フロー:**
1. 投稿詳細画面のコメント入力フォームにテキストを入力する
2. 「コメントする」ボタンをクリックする
3. バックエンドが DB に保存する
4. コメント一覧の末尾に新規コメントが追加表示される
5. 入力フォームがリセットされる

**代替フロー（バリデーションエラー）:**
- テキストが空の場合、「コメントを入力してください」を表示する
- テキストが 280 文字を超えた場合、「280 文字以内で入力してください」を表示する

---

### UC-09: コメント削除

| 項目 | 内容 |
|---|---|
| UC-ID | UC-09 |
| ユースケース名 | コメント削除 |
| アクター | 認証済みユーザー（コメント投稿者本人のみ） |
| 事前条件 | ログイン済み、自分のコメントが存在する |
| 事後条件 | コメントが DB から削除され、一覧から非表示になる |

**基本フロー:**
1. 自分のコメントの「削除」ボタンをクリックする
2. 確認ダイアログが表示される
3. 「削除する」をクリックする
4. バックエンドがコメントを DB から削除する
5. コメント一覧から当該コメントが消える

**代替フロー（キャンセル）:**
- 「キャンセル」をクリックした場合、ダイアログを閉じる

---

### UC-10: コメント数の確認

| 項目 | 内容 |
|---|---|
| UC-ID | UC-10 |
| ユースケース名 | コメント数の確認 |
| アクター | 認証済みユーザー |
| 事前条件 | ログイン済み、タイムライン画面を表示中 |
| 事後条件 | 投稿カードにコメント数が表示される |

**基本フロー:**
1. タイムライン画面を表示する
2. 各投稿カードのコメントアイコンの横にコメント件数が表示される
3. コメントアイコンをクリックすると投稿詳細画面に遷移し、コメント一覧を閲覧できる

---

## 3. 機能要件

### コメント

- [ ] コメント本文は 1〜280 文字とする
- [ ] 投稿者本人・第三者どちらもコメントできる
- [ ] 自分のコメントのみ削除できる（他ユーザーのコメントは削除不可）
- [ ] コメントの編集機能は本フェーズでは実装しない
- [ ] コメントは投稿詳細画面でのみ行える（タイムライン上からは不可）

### コメント一覧表示

- [ ] 投稿詳細画面にてコメントを古い順（created_at ASC）で表示する
- [ ] 各コメントにはアバター・表示名・ユーザー名・コメント日時・本文を表示する
- [ ] 自分のコメントには「削除」ボタンを表示する
- [ ] タイムラインの投稿カードにはコメント件数を表示する

## 4. API 設計（概要）

| メソッド | エンドポイント | 説明 | 認証 |
|---|---|---|---|
| GET | `/api/posts/{postId}/comments` | コメント一覧取得 | 必要 |
| POST | `/api/posts/{postId}/comments` | コメント投稿 | 必要 |
| DELETE | `/api/posts/{postId}/comments/{commentId}` | コメント削除（本人のみ） | 必要 |
