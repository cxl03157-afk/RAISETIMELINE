# 機能定義書：タイムライン・投稿機能

## 1. 業務フロー

### 1-1. タイムライン表示フロー
```
ログイン済みユーザーがホーム画面を開く
    │
タイムライン上部のタブを選択
    ├── 「全体」タブ（デフォルト）→ 全ユーザーの投稿を新着順で取得
    └── 「フォロー中」タブ → フォロー中ユーザーの投稿のみを新着順で取得
    │
タイムラインに投稿カードを一覧表示
    │
    ├── 投稿カードをクリック → 投稿詳細画面へ遷移
    ├── アバターをクリック → 投稿者のプロフィール画面へ遷移
    ├── いいねボタンをクリック → いいね機能（UC参照）
    ├── コメントアイコンをクリック → 投稿詳細画面へ遷移
    └── 「さらに読み込む」をクリック → 次ページの投稿を取得して追加表示
```

### 1-2. 投稿作成フロー
```
「投稿する」ボタンをクリック
    │
投稿作成モーダルを表示
    │
テキストを入力（最大 280 文字）
    │
    ├── 画像を添付する場合 → 画像を選択（最大 4 枚） ※詳細は 05_image.md 参照
    │
「投稿」ボタンをクリック
    │
    ├── テキストが空かつ画像なし → 「投稿内容を入力してください」を表示
    │
    └── バリデーション成功
            │
        バックエンドに投稿データを送信
            │
        モーダルを閉じる
            │
        タイムライン先頭に新規投稿を表示
```

### 1-3. 投稿編集フロー
```
自分の投稿カードの「…」メニューを開く
    │
「編集」を選択
    │
投稿編集モーダルを表示（現在のテキスト・画像がプリセット）
    │
テキストを修正（画像の追加・削除も可）
    │
「更新」ボタンをクリック
    │
    ├── テキストが空かつ画像なし → 「投稿内容を入力してください」を表示
    │
    └── バリデーション成功
            │
        バックエンドに更新リクエストを送信（updated_at を更新）
            │
        モーダルを閉じる
            │
        タイムライン上の投稿を更新表示
```

### 1-4. 投稿削除フロー
```
自分の投稿カードの「…」メニューを開く
    │
「削除」を選択
    │
確認ダイアログ「この投稿を削除しますか？」を表示
    │
    ├── 「キャンセル」→ ダイアログを閉じる
    │
    └── 「削除する」→ バックエンドに削除リクエストを送信
                        │
                    タイムラインから投稿を非表示にする
```

## 2. ユースケース

### UC-05: タイムライン表示

| 項目 | 内容 |
|---|---|
| UC-ID | UC-05 |
| ユースケース名 | タイムライン表示 |
| アクター | 認証済みユーザー |
| 事前条件 | ログイン済み |
| 事後条件 | 選択タブに応じた投稿が新着順に表示される |

**基本フロー:**
1. ユーザーがホーム（タイムライン）画面に遷移する
2. デフォルトで「全体」タブが選択される
3. バックエンドから最新 20 件の投稿を取得する（全体 or フォロー中）
4. 各投稿カードにアバター・表示名・ユーザー名・投稿日時・本文・画像（あれば）・いいね数・コメント数を表示する
5. 「フォロー中」タブをクリックするとフォロー中ユーザーの投稿のみ切り替え表示する
6. 「さらに読み込む」ボタンで次ページを追加取得する

### UC-06: 投稿作成

| 項目 | 内容 |
|---|---|
| UC-ID | UC-06 |
| ユースケース名 | 投稿作成 |
| アクター | 認証済みユーザー |
| 事前条件 | ログイン済み |
| 事後条件 | 投稿が DB に保存され、タイムラインに表示される |

**基本フロー:**
1. 「投稿する」ボタンをクリックする
2. 投稿作成モーダルが開く
3. テキストを入力する（必要に応じて画像を添付）
4. 「投稿」ボタンをクリックする
5. バックエンドが DB に保存する
6. モーダルが閉じ、タイムライン先頭に新規投稿が表示される

**代替フロー（バリデーションエラー）:**
- テキストが空かつ画像が未添付の場合、「投稿内容を入力してください」を表示する
- テキストが 280 文字を超えた場合、「280 文字以内で入力してください」を表示する

### UC-06b: 投稿編集

| 項目 | 内容 |
|---|---|
| UC-ID | UC-06b |
| ユースケース名 | 投稿編集 |
| アクター | 認証済みユーザー（投稿者本人のみ） |
| 事前条件 | ログイン済み、自分の投稿が存在する |
| 事後条件 | 投稿内容が DB に更新され、タイムラインに反映される |

**基本フロー:**
1. 自分の投稿の「…」メニューをクリックする
2. 「編集」を選択する
3. 投稿編集モーダルが開く（現在のテキスト・画像がプリセット）
4. テキストを修正する（必要に応じて画像を追加・削除）
5. 「更新」ボタンをクリックする
6. バックエンドが DB を更新する（`updated_at` も更新）
7. モーダルが閉じ、タイムライン上の投稿が更新表示される

**代替フロー（バリデーションエラー）:**
- テキストが空かつ画像が未添付の場合、「投稿内容を入力してください」を表示する
- テキストが 280 文字を超えた場合、「280 文字以内で入力してください」を表示する

**代替フロー（キャンセル）:**
- 「×」をクリックした場合、変更を破棄してモーダルを閉じる

---

### UC-07: 投稿削除

| 項目 | 内容 |
|---|---|
| UC-ID | UC-07 |
| ユースケース名 | 投稿削除 |
| アクター | 認証済みユーザー（投稿者本人のみ） |
| 事前条件 | ログイン済み、自分の投稿が存在する |
| 事後条件 | 投稿・関連コメント・いいね・画像が DB から削除される |

**基本フロー:**
1. 自分の投稿の「…」メニューをクリックする
2. 「削除」を選択する
3. 確認ダイアログが表示される
4. 「削除する」をクリックする
5. バックエンドが投稿・関連データ（コメント・いいね・画像）をカスケード削除する
6. タイムラインから当該投稿が消える

**代替フロー（キャンセル）:**
- 「キャンセル」をクリックした場合、ダイアログを閉じる

## 3. 機能要件

### 投稿

- [ ] 投稿本文は 1〜280 文字とする
- [ ] テキストのみ・画像のみ・テキスト＋画像のいずれかで投稿できる
- [ ] 自分の投稿のみ編集・削除できる（他ユーザーの投稿は操作不可）
- [ ] 投稿削除時に関連コメント・いいね・画像（S3 ファイル含む）を連動削除する
- [ ] 投稿編集時に `updated_at` を更新する

### タイムライン表示

- [ ] タイムライン上部に「全体」「フォロー中」のタブを表示する
- [ ] 「全体」タブ：全ユーザーの投稿を新着順（created_at DESC）で表示する（デフォルト）
- [ ] 「フォロー中」タブ：ログインユーザーがフォローしているユーザーの投稿のみ表示する
- [ ] 1 ページあたり 20 件を表示し、「さらに読み込む」でページネーションを行う
- [ ] 各投稿カードには以下を表示する：
  - アバター画像（未設定時はデフォルト画像）
  - 表示名・ユーザー名（@username）
  - 投稿日時（例：2026-05-18 15:30）
  - 投稿本文
  - 添付画像（最大 4 枚グリッド表示）
  - いいね数・コメント数
  - 自分の投稿には「…」メニュー（編集・削除）を表示する

## 4. API 設計（概要）

| メソッド | エンドポイント | 説明 | 認証 |
|---|---|---|---|
| GET | `/api/posts?page=0&size=20` | タイムライン取得・全体（ページネーション） | 必要 |
| GET | `/api/posts/following?page=0&size=20` | タイムライン取得・フォロー中（ページネーション） | 必要 |
| POST | `/api/posts` | 投稿作成 | 必要 |
| PUT | `/api/posts/{postId}` | 投稿編集（本人のみ） | 必要 |
| DELETE | `/api/posts/{postId}` | 投稿削除（本人のみ） | 必要 |
| GET | `/api/posts/{postId}` | 投稿詳細取得 | 必要 |
| GET | `/api/users/{username}/posts` | ユーザー別投稿一覧 | 必要 |
