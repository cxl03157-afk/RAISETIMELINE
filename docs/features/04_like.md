# 機能定義書：いいね機能

## 1. 業務フロー

### 1-1. いいね追加フロー
```
タイムラインまたは投稿詳細画面でハートアイコンをクリック
    │
    ├── 未いいね状態の場合
    │       │
    │   バックエンドにいいねを送信・DB に保存
    │       │
    │   ハートアイコンをアクティブ状態（赤）に変更
    │       │
    │   いいね数を +1 して表示を更新
    │
    └── いいね済み状態の場合 → いいね取り消しフローへ
```

### 1-2. いいね取り消しフロー
```
いいね済みのハートアイコン（赤）をクリック
    │
バックエンドにいいね取り消しを送信・DB から削除
    │
ハートアイコンを非アクティブ状態（グレー/アウトライン）に変更
    │
いいね数を -1 して表示を更新
```

## 2. ユースケース

### UC-11: いいね

| 項目 | 内容 |
|---|---|
| UC-ID | UC-11 |
| ユースケース名 | いいね |
| アクター | 認証済みユーザー（投稿者本人・第三者どちらも可） |
| 事前条件 | ログイン済み、対象投稿が未いいね状態 |
| 事後条件 | いいねが DB に保存され、いいね数が +1 される |

**基本フロー:**
1. タイムラインまたは投稿詳細画面でハートアイコンをクリックする
2. バックエンドが likes テーブルにレコードを追加する
3. ハートアイコンがアクティブ状態（赤）に変わる
4. いいね数が +1 されて表示が更新される

**代替フロー（二重いいね防止）:**
- 既にいいね済みの投稿に対して再度いいねをリクエストした場合、バックエンドは 409 Conflict を返す
- フロントエンドは UI 状態とサーバー状態の整合を保つ（楽観的更新後にロールバック）

---

### UC-12: いいね取り消し

| 項目 | 内容 |
|---|---|
| UC-ID | UC-12 |
| ユースケース名 | いいね取り消し |
| アクター | 認証済みユーザー（いいねを行ったユーザー本人のみ） |
| 事前条件 | ログイン済み、対象投稿をいいね済み |
| 事後条件 | いいねが DB から削除され、いいね数が -1 される |

**基本フロー:**
1. いいね済みのハートアイコン（赤）をクリックする
2. バックエンドが likes テーブルからレコードを削除する
3. ハートアイコンが非アクティブ状態（アウトライン）に変わる
4. いいね数が -1 されて表示が更新される

---

### UC-13: いいね数の確認

| 項目 | 内容 |
|---|---|
| UC-ID | UC-13 |
| ユースケース名 | いいね数の確認 |
| アクター | 認証済みユーザー |
| 事前条件 | ログイン済み、タイムライン画面または投稿詳細画面を表示中 |
| 事後条件 | 投稿カードにいいね数とログインユーザーのいいね済み状態が表示される |

**基本フロー:**
1. タイムライン画面を表示する
2. 各投稿カードのハートアイコン横にいいね件数が表示される
3. ログインユーザーがいいね済みの投稿はハートアイコンがアクティブ状態（赤）で表示される

---

## 3. 機能要件

### いいね

- [ ] 投稿者本人・第三者どちらもいいねできる
- [ ] 同一ユーザーが同一投稿に複数回いいねできない（DB UNIQUE 制約で保証）
- [ ] いいね / 取り消しはトグル操作で行う（ハートアイコンのクリック）
- [ ] いいね操作はタイムライン画面・投稿詳細画面どちらからでも行える

### 表示

- [ ] タイムラインの投稿カードにいいね件数を表示する
- [ ] ログインユーザーがいいね済みの投稿はハートアイコンをアクティブ状態（赤）で表示する
- [ ] いいね操作時の UI 更新は楽観的更新（Optimistic Update）で即時反映する
